Test 5: Deep Reactivity Unique Feature

            
Changes to nested properties automatically trigger updates. This is unique to this plugin!

            
            

                Deep Reactivity: Unlike basic property binding, this plugin uses deepRead() to subscribe to ALL nested signal changes. When $colorConfig.r changes, the entire object binding updates.
            

            
            

                
                
            

            

                
                
            

            

                
                
            

            

                
                
            

            
            

                
Color Preview

                

                    

                    
                

                
Object Value

                
            

            
            

                Pass to a web component: <color-picker data-prop:color-config="$test5.colorConfig">
            

            
            
data-prop:color-config="$test5.colorConfig"
// Nested changes like $test5.colorConfig.r = 128 trigger updates!

Test 7: Kebab-case Conversion Unique Feature

            
Property names are automatically converted from kebab-case to camelCase.

            
            

                Automatic Conversion: data-prop:my-long-property-name becomes myLongPropertyName on the element.
            

            
            

                
                
            

            
            

                
Element Property

                

                
            

            
            
data-prop:my-long-property-name="$test7.myLongPropertyName"
// Sets element.myLongPropertyName (not element['my-long-property-name'])

Test 9: Lit Component Support Unique Feature

            
This plugin automatically calls requestUpdate() for Lit components.

            
            

                Lit Integration: Datastar's reactive proxies maintain the same object reference when internal values change. This normally prevents Lit from detecting changes. This plugin calls requestUpdate(propName) to ensure Lit's lifecycle methods fire correctly.
            

            
            

                
                
            

            
            

                
                
                
            

            
            

                
Config Object

                
            

            
            

                Usage with Lit: <my-lit-component data-prop:config="$test9.config">
            

            
            
// Lit component will receive requestUpdate('config') calls
// ensuring updated() lifecycle fires even when proxy reference doesn't change
